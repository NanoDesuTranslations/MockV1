Width and images

It's been observed that the banner was wider than the image inside. This is because the original nanodesu translations banner images were 1024px wide, and the width of the banner was bootstrap's default 1170px (on large desktops). There are two ways of fixing this; here they are for comparison:

  • We can stretch the image to fit it's container: version A. This is a simple change to the page, but does require that the image be stretched. Depending on the image, there may be visual artifacts.
  • We can rebuild bootstrap to reduce the size of the container on large desktops: version B. For maintainability, this requires at least keeping a config.json file checked in in the project and modifying that config rather than modifying bootstrap itself. The "right" way to do this would involve using the less tool to build a customized bootstrap.min.css and incorporating less into our build process for the site.

Aside from technical considerations, there's an aesthetic consideration. Is it better to read the chapter in the narrower column or in the wider column?
